“It would seem logical to collect and store the password in an object of type java.lang.String. However, here's the caveat: Objects of type String are immutable, i.e., there are no methods defined that allow you to change (overwrite) or zero out the contents of a String after usage. This feature makes String objects unsuitable for storing security sensitive information such as user passwords. You should always collect and store security sensitive information in a char array instead.”

Trilha JavaEE (06/07)

Docker e Java EE

  • Polêmica Java EE e Oracle
  • Docker - container para abstrair o local de deploy (sua máquina ou nuvem)
  • Abstrai o tipo de container (apache, glassfish, weblogic ...)
  • Dá pra configurar o load balancer
  • Ferramentas de orquestração (amazon, softlayer, etc)

CDI 2.0

  • Framework para injeção de dependência (como o spring)
  • Ciclo de vida bem definido
  • Integração com expression language
  • Object decoration / Typesafe Interseptor Binding / Interseptor para Java EE specs
  • Event Notification Model (@Observer, @Priority)

Hibernate 5

  • Standard Service Register Builder mo lugar do SessionFactory
  • Novos tipos do Java 8 (DateTime APIs)
  • Tipos Genéricos do Java 5,6,7 e AutoCloseable (try/catch, não precisa mais dar close explicitamente)
  • ScrollableResults
  • Bytecode augmentation
  • Gerencia automaticamente os 2 lados do relacionamento
  • Attribute converter (Enum sem annotation @Enumerated)

Hibernate 5

  • @Generatedvalue (strategy=auto) pode usar o UUID
  • Hibernate Search (permite consultas textuais, indexar blob (como no google)
  • Backend elastic search, lucene 5, melhorias de performance
  • @Spatial, @INdexed, @Latitude @Logitude [Alô Heroes!]
  • Hibernate OGM (Object Grid Mapper) - para banco nosql
  • Hibernate Envers (para versionamento e auditoria)

Trilha Banco de dados (05/07)

Big data e OLTP juntos: Polybase

  • Windows azure blob storage - o cluster (HD Insight cluster) só serviria para processar os dados e não armazenar, barateando o custo de uso)
  • Polybase
    • Mapeia SQL Server com o ambiente de big data
    • Recentemente disponível em distribuições SQL Server
    • Storage é lido como tabelas
    • Pode se conectar com blob storage sem o cluster estar no ar

Polybase

  • Pode se conectar ao master do hadoop (nesse caso o polybase cria um job - jar - para rodar lá. Isso se chama pushdown)
  • Você configura como será a conversão de dados do storage para SQL (ex mapear CSV para "Create Table")
  • Necessário criar credencial se o storage tiver autenticação kerberos
  • As tabelas criadas são temporárias

Medição e gerenciamento com SQL Server

  • Uso do perfmon (add counters -> physical disk)
  • Configurar o paralelismo do SQL Servercom 'maxdop' (se > 5, é pq usa paralelismo) e 'cost of threshold'(ver relação entre os 2)
  • Por default, o SQL server é configurado para usar toda a memória da máquina
  • O ideal é delimitar (mas requer estudo de como você usa a máquina)
  • SQL monitor

Cobertura de código de procedures TSQL

  • SQLCC - projeto descontinuado
  • Demo no youtube (youtu.be/inZHr63-ec0)
  • Cobertura de testes unitários de TSQL

Influx DB

  • Banco de dados baseado em série temporal
  • Baseado em nosql
  • Outros exemplos: Open TSDB, Atlas (usado pela Netflix), Prometheus
  • Escrta e leitura com REST (não tem update, delete, pq é temporal)
  • Linguagem semelhante ao SQL
  • Permite indexar séries
  • T.I.C.K - faz as consultas InfluxDB e gera relatórios (docs.influxdata.com)
  • Com o Grafana, pode consolidar medições e realizar funções (aggregations, selections, transformations, etc)

Trilha NoSQL (08/07)

No SQL

  • Tutorial 1
  • Tutorial 2
  • Não relacional
  • Baseado em JSON
  • JSON é um tipo em SQL, como INTEGER
  • É possível fazer consultas e updates sobre o JSON

Dynamob em C#

  • Banco como serviço da Amazon
  • Cada linha pode ter um numero diferente de colunas
  • Key/value/column oriented
  • columns-> attribute; row -> item
  • Se repetir o insert, entende como update
  • Consultas realizadas como requisiçãoao server da Amazon
  • Tem que pensar nas queries antes de projetar
  • O join é feito pelo programador (no C#)

Cassandra

  • cassandra.apache.org
  • Schema -> keyspace
  • Table -> Column family
  • Primary keys - row key
  • SQL -> CQL
  • Se usar driver kundera (JPA), pode continuar usando annotations do hibernate

Cassandra

  • Cqlsh - prompt para consultas cassandra
  • trabalha com memtabel e depois salva em disco (operação de flush)
  • Mantém histórico de dados (como DB temporal)
  • Particionamento de dados
    • Cuidado com partições do disco!
    • A linha tem que vaber em uma partição (tem cálculo para isso, que varia)
    • Problemas de partição causam System Failuer (solução: usar buckets)

Cassandra - Modelagem

  • Query driven
    • Uma entidade por consulta
    • Tabelas retornam dados pré ordenados
    • Duplicação de dados - melhor duplicar que fazer join, pois é mais escalável
    • tem tipos multivalorados (collections), mas tem que ser pouscos dados (até 64K)
  • Time series
    • Tem uma rowkey e suas colunas
    • Tem timestamp
    • Para coleta de dados ao longo do tempo
    • armazenado em uma única linha

MongoDB

  • Representação de dados geoespaciais
  • Documentos codificados em binário
  • GeoJSON - pardão aberto para codificar estruturas de dados (points, multipoints, polygons)

NoSQL baseado em grafos

  • TinkerPop - usa linguagem 'Gremlin'
  • Titan - usado para grafos grandes
  • Net4J
  • Permitem cruzamento de dados entre entidades aparentemente não relacionados
  • Rápido processamento
  • Caso Panama Papers
